# Architecture Decision Records (ADR)

Issue対応中に得られた重要な知見を記録するディレクトリです。

## ファイル命名規則

```
NNN-簡潔な問題名.md
```

- `NNN`: 3桁の連番（001, 002, ...）
- 既存ファイルの最大番号 + 1 を使用

## テンプレート

```markdown
# NNN: 問題名 (YYYY-MM-DD)

## 問題
何が起きたか

## 対応
試したこと、最終的な解決策

## 教訓
今後の注意点
```

## 記録すべき内容

1. 外部依存の仕様変更対応
2. 「これはダメだった」という知見
3. 設計上の意図的な制限
4. 環境依存の注意点

## 注意

- このファイルを作成したら、**必ず AGENTS.md の「既知の制約」セクションにも1行サマリーとリンクを追加**すること
- AGENTS.md に書かないと他のエージェントから参照されない
