# Issue #292 実装計画書

## 概要

`agents/merge.md` エージェントテンプレートに「完了報告」セクションが2回記載されている問題を修正する。

## 問題分析

ファイル `agents/merge.md` に以下の2つの「完了報告」セクションが存在：

1. **1つ目（採用予定）**: 詳細な説明と `watch-session.sh` に関する重要な注意事項を含む
2. **2つ目（削除対象）**: 簡略化されたバージョンで、1つ目と情報が重複

## 影響範囲

- `agents/merge.md` のみ
- ドキュメントの修正であり、機能への影響なし

## 実装ステップ

1. 2つ目の「完了報告」セクションを削除
2. 必要に応じて1つ目のセクションの内容を確認・調整
3. 変更をコミット

## テスト方針

- `grep -c "完了報告" agents/merge.md` で確認（結果が1になること）
- ファイルの構造が正しいことを目視確認

## リスクと対策

- **リスク**: 必要な情報が削除される可能性
- **対策**: 1つ目のセクションが完全な情報を含んでいることを確認

## 削除対象

ファイル末尾の以下のセクション全体を削除：

```markdown
## 完了報告

タスクが正常に完了したら（PRがマージされた後）、以下のマーカーを出力してください：

\`\`\`
###TASK_COMPLETE_{{issue_number}}###
\`\`\`

これにより、worktree と tmux セッションが自動的にクリーンアップされます。
```
