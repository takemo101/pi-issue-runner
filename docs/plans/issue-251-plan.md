# Issue #251 実装計画書

## 概要

README.md のテンプレート変数リストを更新し、すべての利用可能な変数を文書化する。

## 問題分析

現在の README.md には以下の3つの変数のみが記載されている:
- `{{issue_number}}`
- `{{branch_name}}`
- `{{worktree_path}}`

しかし、`lib/workflow.sh` (行 175-186) では以下の6つの変数がサポートされている:
1. `{{issue_number}}` - Issue番号
2. `{{issue_title}}` - Issueタイトル（**欠落**）
3. `{{branch_name}}` - ブランチ名
4. `{{worktree_path}}` - ワークツリーパス
5. `{{step_name}}` - 現在のステップ名（**欠落**）
6. `{{workflow_name}}` - ワークフロー名（**欠落**）

## 影響範囲

- `README.md` - テンプレート変数の説明箇所（約220行目付近）

## 実装ステップ

1. README.md の該当箇所を特定
2. テンプレート変数の説明をテーブル形式に変更
3. AGENTS.md との整合性を確認
4. 変更をコミット

## テスト方針

- ドキュメントの変更のみのため、テストコードの追加は不要
- AGENTS.md との整合性を目視確認

## リスクと対策

- **リスク**: 特になし（ドキュメント変更のみ）
- **対策**: AGENTS.md の形式に合わせて整合性を保つ
