# 実装計画書: Issue #202

## 概要

lib/ディレクトリ内のライブラリファイルから不要な実行権限を削除する。これらのファイルは`source`で読み込まれるライブラリであり、直接実行は想定されていない。

## 影響範囲

### 変更対象ファイル
- `lib/config.sh` - 実行権限を削除
- `lib/github.sh` - 実行権限を削除
- `lib/notify.sh` - 実行権限を削除
- `lib/status.sh` - 実行権限を削除
- `lib/tmux.sh` - 実行権限を削除
- `lib/workflow.sh` - 実行権限を削除
- `lib/worktree.sh` - 実行権限を削除

### 変更しないファイル
- `lib/log.sh` - すでに正しく設定されている（実行権限なし）
- `scripts/*.sh` - 直接実行するスクリプトのため実行権限を維持

## 実装ステップ

1. 対象ファイルの現在の権限を確認
2. `chmod -x` で実行権限を削除
3. 変更後の権限を確認
4. テスト実行で既存機能に影響がないことを確認

## テスト方針

1. `./scripts/test.sh` で全テストを実行
2. 権限変更後も`source`による読み込みが正常に動作することを確認
3. `scripts/`の実行権限が維持されていることを確認

## リスクと対策

### リスク
- なし（ファイル内容の変更はない）

### 対策
- 変更前後で`ls -la`により権限を確認
- テスト実行で機能に影響がないことを確認
