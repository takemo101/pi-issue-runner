# Issue #109 実装計画

## 概要

`agents/merge.md`の不要なクリーンアップセクションを修正する。

## 問題点

自動クリーンアップ機能（`scripts/cleanup.sh`）の追加後、以下の手順が不要かつ問題を引き起こす：

1. `git checkout main` - worktree内ではmainが既に親リポジトリでチェックアウトされているため失敗
2. `git branch -d {{branch_name}}` - 自動クリーンアップで対応済み

## 影響範囲

- `agents/merge.md` のみ

## 実装方針

「### 4. クリーンアップ（オプション）」セクションを削除し、自動クリーンアップについての注記に置き換える。

## 変更内容

**Before:**
```markdown
### 4. クリーンアップ（オプション）
マージ後、ローカル環境をクリーンアップします：

```bash
# メインブランチに戻る
git checkout main
git pull

# ローカルブランチを削除
git branch -d {{branch_name}}
```
```

**After:**
```markdown
### 4. クリーンアップ

> **Note**: Worktreeのクリーンアップは自動的に行われます。
> 手動でクリーンアップが必要な場合は `scripts/cleanup.sh` を使用してください。
```

## テスト方針

- ドキュメント変更のため、構文チェック程度
- 自動クリーンアップ機能との整合性確認

## リスクと対策

- 低リスク：ドキュメントのみの変更
