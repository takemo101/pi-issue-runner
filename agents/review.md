実装をセルフレビューします。`gh pr create` はこのステップでは実行しないでください。

#### 1. 変更内容の確認
```bash
git diff main...HEAD --stat
git diff main...HEAD --name-only
```

#### 2. レビュー観点

**🚨 意図しない変更の検出（最優先）**
```bash
git diff main...HEAD --numstat | awk '$2 > $1 * 5 && $2 > 50 {print "⚠️ 大幅削減:", $3, "(+" $1, "-" $2 ")"}'
```
- [ ] Issueの要件に無関係なファイルが変更されていないか
- [ ] ファイルが意図せずスタブに置き換えられていないか

**🔗 波及影響（最重要）**
- [ ] 変更した関数の**出力フォーマット**は呼び出し元の期待と一致しているか
- [ ] 新しいリソースを作成する場合、**cleanup/teardown処理**も対応しているか
- [ ] 既存のインターフェース（引数の数、戻り値の意味）を変更していないか
- [ ] `set -e` 環境で `return 1` がプロセス死亡を招かないか

**🔧 コード品質**
- [ ] エラーハンドリングは適切か
- [ ] 不要なコードやコメントがないか

**🧪 テスト**
- [ ] 変更した機能のテストが書かれているか
- [ ] エッジケース（空文字列、特殊文字）がテストされているか

**🔒 セキュリティ**
- [ ] 外部入力がコマンド実行に渡される箇所はサニタイズされているか

#### 3. 問題の修正
発見した問題を修正し、再コミットしてください。

#### 4. 全テスト再実行
修正後、全テストスイートがパスすることを確認してください。

#### 完了条件
- [ ] 無関係なファイルの変更がないことを確認した
- [ ] 全レビュー観点をチェックした
- [ ] 発見した問題を修正した
- [ ] 全テストスイートがパスした
